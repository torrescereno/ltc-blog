Hay dos preguntas que se repiten en frontend: *¿uso Flexbox o Grid?* y *¿por qué no me centra este div?*

La respuesta corta: Flexbox para una dimensión, Grid para dos. Pero la respuesta útil es saber leer el problema y elegir la herramienta correcta sin dudar.

---

## [Flexbox](https://developer.mozilla.org/es/docs/Learn/CSS/CSS_layout/Flexbox)

Flexbox trabaja en una dirección a la vez: horizontal **o** vertical. Es el modelo perfecto cuando tienes elementos en fila o en columna y necesitas controlar cómo se distribuyen y alinean entre sí.

<Callout type="tip">
  **Dónde brilla Flexbox:** navbars, grupos de botones, cards alineadas, cualquier componente donde los elementos se distribuyen en una sola dirección.
</Callout>

### Las propiedades que realmente usas

```css
.nav {
  display: flex;

  /* Dirección del eje principal */
  flex-direction: row; /* row | row-reverse | column | column-reverse */

  /* Distribución en el eje principal */
  justify-content: space-between; /* flex-start | center | space-around | space-evenly */

  /* Alineación en el eje cruzado */
  align-items: center; /* flex-start | flex-end | stretch | baseline */

  /* Espacio entre elementos (sin márgenes individuales) */
  gap: 16px;
}
```

### Ejemplo real: navbar con logo y links

```html
<nav class="nav">
  <a href="/" class="nav__logo">MiBlog</a>

  <ul class="nav__links">
    <li><a href="/posts">Posts</a></li>
    <li><a href="/about">Sobre mí</a></li>
  </ul>

  <button class="nav__cta">Suscribirse</button>
</nav>
```

```css
.nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 0 24px;
  height: 64px;
  border-bottom: 1px solid #e5e7eb;
}

.nav__links {
  display: flex;      /* Flexbox dentro de Flexbox, totalmente válido */
  gap: 24px;
  list-style: none;
  margin: 0;
  padding: 0;
}
```

### Controlar cómo crecen los elementos

Cuando el contenedor tiene espacio sobrante, `flex-grow` decide quién lo toma:

```css
.sidebar {
  flex: 0 0 260px;   /* no crece, no encoge, siempre 260px */
}

.main-content {
  flex: 1;           /* toma todo el espacio restante */
}
```

`flex: 1` es shorthand de `flex-grow: 1; flex-shrink: 1; flex-basis: 0`. Es el más usado y el que deberías memorizar.

## [Grid](https://developer.mozilla.org/es/docs/Web/CSS/CSS_grid_layout)

Grid trabaja en filas **y** columnas al mismo tiempo. Es el primer sistema CSS diseñado específicamente para layouts de página, y se nota.

<Callout type="info">
  **Dónde brilla Grid:** layouts de página completa, galerías de imágenes, dashboards, cualquier estructura donde los elementos necesitan alinearse en dos ejes.
</Callout>

### Definir la cuadrícula

```css
.layout {
  display: grid;

  /* 3 columnas: sidebar fijo, contenido flexible, aside fijo */
  grid-template-columns: 240px 1fr 200px;

  /* Filas: header fijo, contenido flexible, footer fijo */
  grid-template-rows: 64px 1fr 48px;

  /* La unidad fr = fracción del espacio disponible */
  gap: 24px;

  min-height: 100vh;
}
```

### Nombrar áreas: la killer feature de Grid

En lugar de coordenadas, puedes asignarle un nombre a cada zona del layout:

```css
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 64px 1fr 48px;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  gap: 16px;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

```html
<div class="layout">
  <header class="header">Header</header>
  <aside class="sidebar">Sidebar</aside>
  <main class="main">Contenido principal</main>
  <footer class="footer">Footer</footer>
</div>
```

El layout completo en 10 líneas de CSS. Sin `position: absolute`, sin hacks.

### Galerías responsivas sin media queries

`repeat` + `auto-fill` + `minmax` es una de las combinaciones más poderosas de CSS moderno:

```css
.gallery {
  display: grid;

  /*
    auto-fill: crea tantas columnas como quepan
    minmax(280px, 1fr): cada columna tiene mínimo 280px y máximo 1fr
    Resultado: el grid se adapta solo al ancho del contenedor
  */
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 24px;
}
```

Sin una sola media query, tienes una galería que pasa de 1 a 4 columnas según el ancho disponible.

## Propiedades que se confunden seguido

### `gap` reemplaza a `grid-gap`

`grid-gap` ya está deprecado. Usa `gap` directamente, funciona igual en Flexbox y Grid:

```css
/* Antiguo */
.grid { grid-gap: 20px; }

/* Correcto */
.flex { gap: 16px; }
.grid { gap: 24px; }

/* También puedes separar fila y columna */
.grid { gap: 16px 24px; } /* row-gap column-gap */
```

### `align-items` vs `justify-content`

La confusión más común. Depende del eje principal:

```css
/* En Flexbox con flex-direction: row (default) */
.flex {
  justify-content: center; /* centra horizontalmente */
  align-items: center;     /* centra verticalmente */
}

/* Centrar un div, la forma correcta */
.centered {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

## ¿Cuándo usar cada uno?

| Situación | Herramienta |
|-----------|-------------|
| Navbar, barra de herramientas | Flexbox |
| Grupo de botones o chips | Flexbox |
| Layout completo de página | Grid |
| Galería de cards | Grid |
| Centrar un elemento | Flexbox |
| Dashboard con múltiples secciones | Grid |
| Componente con eje único | Flexbox |

<Callout type="warning">
  **Lo que hacen todos los seniors:** combinan ambos. Grid para el layout macro de la página, Flexbox para los componentes dentro de cada celda. No es uno u otro, son capas.
</Callout>
